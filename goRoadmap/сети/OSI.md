#networking #osi #fundamentals

Модель OSI (Open Systems Interconnection) — концептуальная модель сетевого взаимодействия. Источник: **ISO/IEC 7498-1:1994**.

---

## 7 уровней модели

```
Отправитель                         Получатель
┌──────────────────┐             ┌──────────────────┐
│ 7. Application   │◄──────────►│ 7. Application   │
│ 6. Presentation  │             │ 6. Presentation  │
│ 5. Session       │             │ 5. Session       │
│ 4. Transport     │             │ 4. Transport     │
│ 3. Network       │◄──────────►│ 3. Network       │
│ 2. Data Link     │◄──────────►│ 2. Data Link     │
│ 1. Physical      │◄──────────►│ 1. Physical      │
└──────────────────┘             └──────────────────┘
```

Уровни 5-7 взаимодействуют **логически**; уровни 1-4 — **физически** (через промежуточные узлы).

---

## Уровни, PDU и протоколы

| # | Уровень | PDU | Протоколы | Адресация |
|---|---|---|---|---|
| 7 | Application | Данные | HTTP, DNS, SMTP, FTP, gRPC | — |
| 6 | Presentation | Данные | TLS/SSL, MIME, JSON/XML кодировки | — |
| 5 | Session | Данные | RPC, NetBIOS, SOCKS | Session ID |
| 4 | Transport | Сегмент / Датаграмма | TCP, UDP, SCTP, QUIC | Port |
| 3 | Network | Пакет | IP (v4/v6), ICMP, OSPF, BGP | IP-адрес |
| 2 | Data Link | Фрейм | Ethernet, Wi-Fi (802.11), PPP | MAC-адрес |
| 1 | Physical | Биты | DSL, 1000BASE-T, USB, Bluetooth | — |

## Инкапсуляция

При отправке каждый уровень добавляет свой **заголовок** (и иногда трейлер):

```
[HTTP данные]
[TCP header | HTTP данные]
[IP header  | TCP header | HTTP данные]
[ETH header | IP header  | TCP header | HTTP данные | ETH trailer]
[биты..................................................................]
```

При получении каждый уровень снимает свой заголовок и передаёт данные выше.

## Разбор уровней

### L7 — Application

Протоколы прикладного уровня определяют **синтаксис и семантику** сообщений.

- **HTTP/1.1** (RFC 7230) — текстовый, request-response, keep-alive
- **HTTP/2** (RFC 7540) — бинарное фреймирование, мультиплексирование, HPACK
- **DNS** (RFC 1035) — разрешение имён; UDP/53 для запросов, TCP/53 для зонных передач
- **SMTP** (RFC 5321) — передача почты; STARTTLS для шифрования

### L6 — Presentation

Трансформация данных: сериализация, шифрование, сжатие.

- **TLS 1.3** (RFC 8446) — шифрование на этом уровне; хэндшейк 1-RTT
- **MIME** — кодирование бинарных данных в текст (Base64)

На практике L6 часто объединяют с L7 — границы размыты.

### L5 — Session

Управление сессиями (установка, поддержание, завершение диалога).

На практике в TCP/IP стеке отдельного L5 нет — его функции поглощены L4 и L7.

### L4 — Transport

Сквозная (end-to-end) передача между процессами.

- **TCP** — надёжность, порядок, управление потоком → [[TCP]]
- **UDP** — без гарантий, минимальный overhead (DNS, видео, игры)
- **SCTP** (RFC 4960) — multi-homing, multi-streaming; используется в телекоме
- **QUIC** (RFC 9000) — надёжность поверх UDP, 0-RTT, встроенный TLS 1.3

### L3 — Network

Маршрутизация пакетов между сетями.

- **IPv4** (RFC 791) — 32-битная адресация, фрагментация
- **IPv6** (RFC 8200) — 128-битная адресация, нет фрагментации на промежуточных узлах
- **ICMP** (RFC 792) — диагностика (`ping`, `traceroute`)
- **BGP** (RFC 4271) — протокол маршрутизации между AS (автономными системами)

### L2 — Data Link

Передача фреймов между соседними узлами в одной сети.

- **Ethernet** (IEEE 802.3) — MAC-адреса, CSMA/CD (устарело с full-duplex)
- **802.11 (Wi-Fi)** — CSMA/CA
- **ARP** (RFC 826) — разрешение IP → MAC (технически L2.5)
- **VLAN** (IEEE 802.1Q) — тегирование фреймов для сегментации L2

### L1 — Physical

Передача битов по физическому носителю: напряжение, свет, радиоволны.

- **1000BASE-T** — гигабитный Ethernet по витой паре
- **Fiber** — одномодовый (SM, ~100 км), многомодовый (MM, ~1 км)

## Модель TCP/IP vs OSI

TCP/IP стек не совпадает с OSI строго:

| TCP/IP | OSI уровни |
|---|---|
| Application | 5, 6, 7 |
| Transport | 4 |
| Internet | 3 |
| Link | 1, 2 |
