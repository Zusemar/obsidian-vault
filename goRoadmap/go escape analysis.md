

**Escape analysis** — это процесс, встраиваемый в компилятор Go, который определяет, должна ли переменная быть выделена на **стеке** (stack) или в **куче** (heap).

---

### Точка входа и Основной Алгоритм

- **Точка входа:** Escape Analysis начинается с вызова функции `escape.Funcs` внутри главной функции компилятора (`/go/src/cmd/compile/internal/gc/main.go`).  
    
- **Callback `Batch`:** Фактическая работа анализа выполняется в функции `Batch` (пакет `cmd/compile/internal/escape`).

#### Этапы работы функции `Batch`:

Функция `Batch` разбивается на три основных этапа:

1. **Построение Графа:** Строится направленный **граф весов (weighted directed graph)**, который моделирует потоки данных и операций с памятью в функции.
2. **Проверка Размера:** Выполняется проверка на соответствие размера переменных установленным лимитам.
3. **Обход Графа:** Осуществляется обход графа (`walkAll`) для вынесения окончательного решения о побеге (escape) каждой переменной в кучу.

---

### Граф Весов (Weighted Graph)

Граф весов — это модель, используемая для отслеживания **отношений между переменными** и **типов операций с указателями**.

- **Ноды (Узлы):** Представляют собой **сами переменные** в функции, включая **псевдопеременные** (например, `r1` для возвращаемого значения).  
    
- **Ребра (Стрелки):** Представляют **операции присваивания** (поток данных).  
    
- **Веса (Weights):** Представляют тип присваивания или операции с указателем:
    - **:** Обычное присваивание (например, `l1 = a`).  
        
    - **:** **Разыменование** указателя (извлечение значения из указателя).  
        
    - **:** Взятие **адреса/ссылки** от переменной (создание указателя, например, `&l3`).  
        

#### Пример:

В графе переменные связаны присваиваниями. Веса показывают, как эти присваивания влияют на "уровень косвенности". Алгоритм (находящийся в файле `cmd/compile/internal/escape/solve.go`) использует этот граф для определения, может ли переменная быть доступна после выхода из области видимости (escape). Если переменная должна быть доступна после завершения функции (например, если она возвращается по ссылке), она "сбегает" в кучу.
